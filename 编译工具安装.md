# Better-SQLite3 编译问题解决方案

## 问题原因
better-sqlite3是原生C++模块，需要Visual Studio构建工具才能编译

## 推荐方案：安装windows-build-tools（自动化）

### 以管理员身份运行PowerShell，执行：

```powershell
# 方法1：使用npm（推荐）
npm install --global windows-build-tools

# 等待10-15分钟安装完成
# 会自动安装：
# - Python 2.7
# - Visual Studio Build Tools 2019
```

### 安装完成后重新安装项目依赖：

```powershell
cd C:\Users\aaron\Desktop\创锦一对一\project\schedule-system
yarn install
```

## 快速方案：使用预编译版本

如果上面的方法太慢，可以使用预编译的better-sqlite3：

```powershell
# 删除现有的node_modules
Remove-Item -Recurse -Force node_modules

# 使用预编译版本安装
yarn add better-sqlite3 --ignore-scripts

# 然后安装其他依赖
yarn install --ignore-scripts
```

## 临时方案：移除better-sqlite3（快速测试）

如果只是想快速看到界面效果，可以暂时移除数据库功能：

1. 打开 package.json
2. 删除 "better-sqlite3": "^9.2.2" 这一行
3. 运行 yarn install
4. 注释掉数据库相关代码（我会帮你处理）

## 验证安装

```powershell
# 检查是否安装成功
node -p "require('better-sqlite3')"

# 如果没报错，说明安装成功
```
